## 새 콘텐츠 흐름 — PreviewCard 기반

이전: ↻ 누르면 3명 풀 콘텐츠 미리 generate (광고 1번 → Gemini 3회). 사용자가 1명만 봐도 2/3 Gemini 비용 손실.

이후: **Daily 카드는 candidate 미리보기만**, 카드 탭 시점에 광고 + (캐시 있으면 fetch / 없으면 generate). 안 본 인물엔 비용 0.

- `0004_candidate_previews.sql`: `figure_candidates`에 `image_url`, `image_credit`, `preview_ko` + categories GIN 인덱스
- `scripts/prefetch-previews.ts`: candidates 전수 Wikipedia 호출 → 사진 Storage mirror + 한국어 summary 첫 문장. Gemini 0원.
- 새 타입 `PreviewCard`, 함수 `loadDailyPreviews()`, 컴포넌트 `PreviewCardView`. strict field filter + 정치/군사 sensitive 제외 + 한국 인물 2× 가중.
- `Figure` 라우트 params를 `{ figureId } | { preview }` 유니온으로. 후자는 진입 시 `generateAndCacheFigure(slug)` 호출.
- `DailyScreen` 전면 리팩터 — `MockAdModal` 5초 후 navigate. 카드 한 번 더 보면 캐시라 즉시.
- ↻는 dev-only로 격하 (광고 X, 셔플만, 현재 표시 슬러그는 viewed mark).

비용 곡선
- 위인 1명 = Gemini ~6원, 한 번만. 그 후 모든 사용자 = 0원.
- 풀 1만명 시점에 캐시 hit 95%+ → 광고 수익이 거의 그대로 이익.

## 갤러리 한국어화 — server build

이전: FigureScreen이 진입 시점에 영문 위키 media-list를 fetch. 영문 캡션 + 부정확한 사진 (코코 샤넬 페이지의 남성 배우 등).

이후: server에서 일괄 빌드 + 영구 저장.

- `scripts/build-galleries.ts`: 한국어 위키 우선, 영문 fallback, Gemini 1회 호출로 캡션 전체를 한국어 (또는 본인 아닌 사진은 SKIP). 결과를 `figures.data.gallery`에 저장.
- 결과: 50명 figures 중 47명 갤러리 빌드, 3명은 위키 미디어 자체 없음. 캡션 자동 검증 (예: einstein 6개 → 5개 keep, marie-curie 3개 → 1개, mlk 5개 → 3개).
- `Gallery` 컴포넌트는 이제 한국어 캡션 없는 카드 자동 hide. 부정확한 사진이 갤러리 안 보이는 게 우선.
- `FigureScreen`의 client-side wiki fallback도 한국어 위키 우선으로 (서버 빌드가 없는 figure 안전망).

## personalize 문구 다시

`comparisonCopy()` 케이스별 재작성:

- later_years 이벤트는 매칭에서 제외 (죽음을 "경험했다"로 표현 회피)
- 같은 나이대(±1): positive 이벤트면 "지금의 당신과 같은 나이에 X의 시기를 보내고 있었습니다", setback이면 "지나고 있었습니다"
- 사용자가 더 나이 많음 + positive: "시작은 어디서든 가능합니다"
- 사용자가 더 나이 많음 + setback: "그 후로도 인생은 길게 이어졌습니다"
- 위인이 더 나이 많음 + positive: "그 시간은 아직 당신에게 남아 있습니다"
- 위인이 더 나이 많음 + setback: "모든 인생에 굴곡이 있습니다"

"준비할 시간 N년 더 길게" 같은 의미 안 닿는 문장 전부 제거.

## 마주친 함정

- supabase-js의 `PostgrestError`는 Error 인스턴스 아님 → `e instanceof Error` false → "[object Object]" UI에. `throw new Error(\`Supabase: ${cErr.message}\`)`로 래핑.
- 0004 SQL 콘솔에서 안 실행하면 클라이언트 진입 즉시 컬럼 not found. 에러 메시지에 ' — 0004 마이그레이션 미적용일 수 있어요' 힌트 추가.
- 마지막 fallback이 viewed slug 제외라 사용자가 풀 다 봤을 때 1~2장만 떠짐 → "viewed 무시" 최후 fallback 추가 (3장 보장 > 노 리피트).
- 새 build → 새 keystore → 폰 install 시 signature mismatch. 첫 install 후로는 무관.
- Gallery는 figure_candidates 사진 prefetch로 미리보기 OK이지만, FigureScreen의 figures.data.gallery는 별도 빌드 필요 (translate-captions 옛 스크립트는 갤러리 0개라 무용 → `build-galleries.ts` 신규).
